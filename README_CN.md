# OpenClaw Agent 使用模式

> 如何有效领导 AI Agent — 来自 Manus 后端负责人的框架

[English](README.md) | 中文

## 三种使用模式

| 模式 | 英文 | 行为 | 结局 |
|------|------|------|------|
| **将才** | Captain | 并肩作战，边用边学 | 成长 |
| **帅才** | Architect | 系统设计，探边界，验质量 | 高效 |
| **刘禅** | Abdicator | 交给AI就不管，接受任何结果 | 认知退化 |

> "前两种是在使用 AI。第三种是被 AI 使用。"

## 模式一：将才 (Captain)

**与 Agent 并肩作战。委托自己能做但选择不做的任务。**

关键洞察：每次委托也是一次学习。不只是得到输出，更得到教育。

### 历史原型
- **韩信** - 从普通士兵起步，成为史上最伟大的将领之一
- **凯撒** - 边打仗边写《高卢战记》

### OpenClaw 用户建议
从这里开始。用 OpenClaw 执行任务，但注意观察它**如何**解决问题。这就是复利所在。

## 模式二：帅才 (Architect)

**不直接干活。把认知精力投入三件事：**

1. **探测 (Probing)** - 系统性地探测 Agent 的能力边界
2. **分解 (Decomposition)** - 把复杂目标拆解成可靠的小单元
3. **验证 (Verification)** - 在关键节点抽查质量

### 历史原型
- **刘邦** - "吾能用之，此吾所以取天下也"
- **艾森豪威尔** - 诺曼底登陆一枪未发，但完成了史上最大两栖入侵

### OpenClaw 用户建议
当已验证的工作流需要跨项目部署时，切换到此模式。设计系统，让正确的 Agent 解决正确的问题。

## 模式三：刘禅 (Abdicator)

**管理学里最脏的词。把任务扔给 Agent，接受任何结果，直接发布。**

不测边界。不查质量。不思考。

### 历史原型
- **刘禅** - "此间乐，不思蜀"
- **尼禄** - 罗马大火时在演奏音乐

### 与 MIT 研究的关联

MIT "认知债务" 论文显示重度 AI 用户的大脑连接减弱。

**但结论不是 "AI 让人变笨"。**

数据表明，**被动消费 AI 输出** 才会削弱认知。对于主动领导 AI 的人，论文没有任何结论。

> AI 没有让任何人变笨。放弃思考才会让人变笨。AI 只是让放弃变得前所未有的容易。

## 是模式，不是类型

**这是三种模式，不是三种人。**

- 探索新工具时，我做将才
- 部署已验证工作流时，我切换到帅才
- 最好的实践者能流畅地切换

韩信和刘邦在同一个故事里。没有对方，谁也赢不了。将才需要帅才的系统。帅才需要将才的前线情报。

## 自检问题

下次启动 OpenClaw（或任何 AI Agent）时，问自己：

> 我现在是将才、帅才，还是刘禅？
