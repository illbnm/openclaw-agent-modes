# OpenClaw Agent 使用模式

## 总览

这个框架帮助你识别自己如何使用 AI Agent，以及你是在成长还是认知退化。

```
┌───────────────────────────────────────────────────────┐
│                  AI AGENT 使用模式                     │
├─────────────────┬─────────────┬───────────────────────┤
│     将才        │    帅才     │       刘禅            │
│    CAPTAIN      │  ARCHITECT  │     ABDICATOR        │
├─────────────────┼─────────────┼───────────────────────┤
│   边做边学      │  设计验证   │   接受就忘           │
│   成长 ↑       │   效率 ↑    │   退化 ↓             │
└─────────────────┴─────────────┴───────────────────────┘
```

## 核心问题

> **我现在是将才、帅才，还是刘禅？**

## 快速判断流程

```
你把任务委托给 AI 了吗？
    │
    ├─ 没有 → 你还没开始用 AI
    │
    └─ 有了 → 你理解 AI 是怎么解决的吗？
              │
              ├─ 不理解 → 刘禅 ⚠️
              │           你被 AI 使用了
              │
              └─ 理解 → 你学到新东西了吗？
                        │
                        ├─ 学到了 → 将才 ✓
                        │           你在成长
                        │
                        └─ 没学到 → 帅才 ✓
                                    你很高效
```

## 模式对比

| 方面 | 将才 Captain | 帅才 Architect | 刘禅 Abdicator |
|------|-------------|---------------|----------------|
| **角色** | 并肩作战 | 系统设计 | 甩手掌柜 |
| **关注** | 学习 | 效率 | 便利 |
| **验证** | 持续 | 关键节点 | 无 |
| **结果** | 成长 | 规模化 | 退化 |
| **认知** | 主动 | 主动 | 被动 |
| **风险** | 过劳 | 过度系统化 | 认知债务 |

## OpenClaw 实战

### 将才模式
```
任务: "帮我写个 Python 脚本"
行为: 观察 AI 怎么解决问题
结果: 学到 Python 模式 + 得到脚本
```

### 帅才模式
```
任务: "自动化每日报告"
行为: 设计工作流，每步验证
结果: 可靠的自动化系统
```

### 刘禅模式 (避免！)
```
任务: "帮我把活干了"
行为: 接受任何结果
结果: 没学到东西，认知退化
```

## MIT 研究的真相

"认知债务" 论文显示重度 AI 用户大脑连接减弱。

**但是：** 数据表明是**被动消费**削弱认知，不是主动领导。

> AI 没有让任何人变笨。放弃思考才会让人变笨。
